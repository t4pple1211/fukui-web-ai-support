# キッチンカーサイト提案メモ

## 目的

友達のキッチンカーについて、初めて見る人が「何を売っているか」「いつどこで買えるか」「イベント出店を相談できるか」をすぐ理解できるサイトを作る。

Instagramだけだと投稿が流れてしまい、メニュー、価格、出店予定、問い合わせ先が探しにくくなる。サイトを作ることで、必要な情報を1か所にまとめ、SNSからの受け皿にできる。

## プロトタイプ

以下のファイルをブラウザで開くと、サンプルサイトを確認できる。

```text

```

## サイトでできること

### メニューと価格を分かりやすく見せる

お客様が事前に「何が食べられるか」「いくらくらいか」を確認できる。

これにより、初めて来る人の不安を減らし、注文しやすくできる。

### 出店予定をまとめる

「今日どこにいる？」「次はいつ来る？」をサイト上で確認できるようにする。

Instagramの固定投稿やプロフィールリンクからサイトに誘導すると、情報を探す手間が減る。

### イベント出店の相談窓口を作る

企業イベント、地域祭り、マルシェ、学校行事などの出店相談を受けやすくする。

フォーム、Instagram DM、LINE、メールなど、相手が使いやすい連絡方法にできる。

### お店のこだわりを伝える

食材、味、店主の思い、出店スタイルを短く載せることで、価格だけで比較されにくくなる。

「この人のお店で買いたい」と思ってもらうための信頼づくりに役立つ。

### GoogleマップやSNSとつなげる

必要に応じて、Instagram、Googleマップ、LINE、イベント出店相談フォームへのリンクを設置できる。

## 友達に伝える説明

このサイトは、Instagramの代わりではなく、Instagramを見た人が詳しく確認するための場所として使う。

たとえば、InstagramプロフィールにサイトURLを置くと、見に来た人が以下をすぐ確認できる。

* メニュー
* 価格
* 出店予定
* イベント出店の相談方法
* お店の雰囲気

まずは1ページの小さなサイトで十分。運用しながら、写真、メニュー、出店予定、お客様の声を増やしていく。

## 最初に作る範囲

初回実績として作るなら、最初は以下の範囲が現実的。

* トップページ1ページ
* メニュー紹介
* 出店予定
* お店のこだわり
* Instagramへのリンク
* 問い合わせ導線
* スマホ対応

ブログ、決済、更新・管理機能などは最初から入れなくてよい。

## 採用理由

まずは低コストな1ページサイトを採用する。

理由は、初回制作では更新機能や管理画面よりも、まず「見せられる形」を早く作ることが大事だから。キッチンカーの場合、最初に必要なのは複雑な機能ではなく、メニュー、出店予定、問い合わせ先がスマホで分かりやすいこと。

更新画面つきのサイトにする案もあるが、初期段階では管理や更新の手間が増える。出店予定の更新頻度が高くなり、本人が自分で更新したい段階になってから検討すればよい。

## 代替案

### Instagram中心で運用する

写真投稿や日々の告知には強い。すでにフォロワーがいる場合は有効。

ただし、投稿が流れるため、メニュー、価格、出店予定、問い合わせ先を毎回探す必要がある。サイトはInstagramの弱い部分を補う役割にする。

### 自分で更新できるサイトにする

本人が頻繁に出店予定やブログを更新したい場合に向いている。

ただし、保守、セキュリティ、更新画面の使い方が必要になる。最初の実績作りでは、低コストな1ページサイトで始める方が負担が少ない。

## 公開後にかかる費用の考え方

サイトは、作った後に「ネット上に置いておく費用」や「自分のお店用のURLを持つ費用」がかかる場合がある。

難しい言葉で説明するより、まずは「どこまで自分でやりたいか」で考える。

### 選び方

| やりたいこと | 月額の目安 | 向いている形 |
| --- | ---: | --- |
| まずはメニューと出店予定を見せたい | 0円〜数百円程度 | 低コストな1ページサイト |
| 自分で出店予定やお知らせを更新したい | 700円〜2,000円程度 | 更新画面つきのサイト |
| イベント出店の相談を整理したい | 1,500円〜数千円以上 | 出店依頼フォームや管理機能つきのサイト |

金額は2026年7月時点の一般的な目安。実際に契約する前に、使うサービスの料金を確認する。

### 1. まずは低コストで始める

月額費用をできるだけ抑えて、メニュー、出店予定、問い合わせ先を見せる形。

向いているケース:

* まずサイトを持ってみたい
* Instagramから詳しい情報を見せたい
* 更新はたまにでよい
* 出店予定の変更は、こちらに依頼してもよい

費用感:

* サイトの置き場所: 0円で始められる場合がある
* お店専用URLを使う場合: 年額1,500円〜5,000円程度

説明例:

「まずは毎月の負担をほぼ増やさず、メニューと出店予定が見られるページを作る形です。自分で頻繁に更新する機能は少ないですが、最初の一歩としては一番始めやすいです。」

### 2. 自分で出店予定を更新したい

スマホやパソコンから、自分でお知らせや出店予定を変更できる形。

向いているケース:

* 出店場所が頻繁に変わる
* 毎回こちらに更新依頼するのは面倒
* お知らせやブログも自分で投稿したい

費用感:

* サイトの置き場所: 月額500円〜1,500円程度
* お店専用URL: 年額1,500円〜5,000円程度
* 合計目安: 月額700円〜2,000円程度

注意点:

* 更新画面の使い方を覚える必要がある
* 不具合やセキュリティ対策のため、定期的なメンテナンスが必要

説明例:

「出店予定を自分で変えたいなら、更新画面つきにできます。その分、月額費用と管理の手間は少し増えます。」

### 3. イベント出店の相談を受けたい

サイトからイベント出店の相談を受けて、管理しやすくする形。

向いているケース:

* イベント出店の依頼を整理したい
* 問い合わせ内容を後から確認しやすくしたい
* 希望日時、場所、予算、電源の有無などを事前に聞きたい

費用感:

* サイトの置き場所や外部サービス: 月額1,000円〜5,000円以上
* お店専用URL: 年額1,500円〜5,000円程度
* 合計目安: 月額1,500円〜数千円以上

注意点:

* 初期制作費も運用費も上がりやすい
* お客様の名前や連絡先を扱うため、管理方法をきちんと決める必要がある

説明例:

「通常営業の注文受付は今はいらない。イベント出店の相談だけ受けやすくしておけば十分です。最初はInstagram DMやGoogleフォームでもよく、依頼が増えてから専用フォームにしても大丈夫です。」

### 初回提案としてのおすすめ

最初は「低コストな1ページサイト + 必要ならお店専用URL」がおすすめ。

理由は、月額費用を抑えながら、メニュー、出店予定、問い合わせ導線を用意できるため。反応を見てから、自分で更新したいか、イベント出店相談フォームが必要かを判断する方が無駄が少ない。

## リスク・注意点

* 写真の品質で印象が大きく変わる
* メニューや価格が変わる場合、更新方法を決めておく必要がある
* 架空の内容を実績のように見せない
* 営業許可、出店条件、アレルギー表記などは必要に応じて確認する
* 問い合わせフォームを使う場合は、送信先と個人情報の扱いを決める
